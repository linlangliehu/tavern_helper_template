# Phase97 双轴判定验收清单

本清单用于验证神秘复苏模拟器的“厉鬼恐怖程度 vs 灾害/影响范围等级”双轴规则。验收时只检查模型输出是否符合规则，不要求新增 schema 字段。

## 通用检查点

- 正文、短标签、行动建议只写玩家可见信息，不泄露隐藏规律、关键生路、源头位置或后期档案。
- 每轮结算后 `<UpdateVariable>` 必须 replace `/最近行动判定`。
- 涉及规律推理时，`/规律推理记录/-` 必须是对象，并包含 `时间点`、`行为`、`观察结果`、`推断`、`确认等级`、`是否触发规律`、`风险等级`。
- `<choices>`、`<sp_choices>`、【推演选项：】三处 A/B/C/D 必须一致，风险值能追溯到判定路由和玩家可见依据。
- C/B/A/S 只能影响范围、封锁、总部响应和社会压力，不能直接当作对抗强度。
- 不写不存在的 `/当前灵异事件/恐怖程度` 路径；可见恐怖程度判断应写入 `/最近行动判定`、`/规律推理记录`、`/当前灵异事件/可见摘要` 或隐藏档案。

## 6 个基础场景

### 1. 日常调查
输入建议：玩家远距离观察一栋疑似闹鬼的教学楼，只查监控和询问保安，不进入楼内，不触碰可疑媒介。

预期：走调查判定或无需正式判定；死亡风险应为 0-3 或保持不变；不得直接公开完整杀人规律；至少推进线索或可见情报。

### 2. 接触媒介
输入建议：玩家在现场接起不断响起的旧电话，并听到电话另一端传来重复敲门声。

预期：走媒介判定；死亡风险通常 +5 到 +12，若电话明显疑似杀人条件则 +12 到 +25；`/最近行动判定.触发项` 应写电话、声音或疑似媒介；规律确认最多到疑似或部分确认。

### 3. 进入鬼域或异常空间
输入建议：玩家走进一条反复回到原点的走廊，手机无信号，出口位置发生变化。

预期：走鬼域判定；若只是初步异常，写疑似鬼域或异常空间，不直接判完整鬼域核心；死亡风险通常 +8 到 +20；短标签可用 `<sp_location_explore>`。

### 4. 主动用鬼
输入建议：玩家动用自身厉鬼能力强行压制门后的脚步声，试图争取十秒撤离窗口。

预期：走用鬼判定；复苏风险通常 +10 到 +20，高恐怖程度压制可 +20 到 +35；结果只能是短暂窗口、压制失败、反噬或撤离，不得写成消灭厉鬼。

### 5. 使用鬼烛/黄金
输入建议：玩家点燃鬼烛并用黄金袋隔离一张疑似媒介照片，然后观察厉鬼反应。

预期：走灵异物品/压制辅助判定；即时死亡风险可降低或受控，但必须记录鬼烛/黄金资源状态、消耗或副作用；不能说黄金杀死厉鬼。

### 6. 命中杀人规律
输入建议：玩家已经知道敲门声有危险，但仍无保护地回应门外的敲门声。

预期：直接裁定优先；不得用普通随机成功率绕过；死亡风险增量通常不低于 +25，严重时濒死或死亡；若有替死/鬼烛才允许资源救命并记录代价。

## 2 个双轴校准场景

### A. 低影响范围 + 高恐怖程度
输入建议：自定义厉鬼影响范围为 C 级，只在一间教室内活动；杀人规律是“看见黑板上的自己名字后立即被标记，离开教室也会被追上”。

预期：不能因 C 级影响范围而判成容易压制；应写“影响范围小但恐怖程度高/疑似高危”；正面对抗、身体试探、无保护接触媒介必须高危。

### B. 高影响范围 + 中等单体恐怖程度
输入建议：某事件被评为 S 级，因为城市里大量影子鬼奴扩散；但单个鬼奴只能缓慢跟随并可被黄金短时隔绝。

预期：不能把每个鬼奴都写成源头级无解；灾害等级高来自扩散规模和社会压力；单体对抗风险按可见恐怖程度、媒介、数量和资源判断。

## 开局字段实机检查

- 点击自定义厉鬼 1 右上角 `+` 后，应出现自定义厉鬼 2。
- 每只自定义厉鬼应能填写：名称、可见杀人规律、影响范围等级、恐怖程度校准建议。
- 点击“进入神秘复苏世界”后，输入框文本应包含“默认影响范围等级”和每只鬼的四类输入。
- 删除第二只厉鬼后，第二只厉鬼的输入应清空，`+` 应恢复。

- 场景1（日常调查）：通过。raw message 包含 `<UpdateVariable>`、`/最近行动判定`、`<choices>`、`<sp_choices>`；走调查判定，死亡风险为低增量（+1），只给四楼异常、脚步声、电子干扰等可见线索，未泄露完整隐藏规律。
- 场景2（接触媒介）：通过。走媒介判定；电话/敲门声写入触发项；死亡风险主增量 +15，落在疑似杀人媒介区间；规律确认保持“疑似”，并输出规律推理记录和 A/B/C/D。
- 场景3（进入鬼域或异常空间）：通过。输出 `<sp_location_explore>`，写“疑似鬼域（空间侵蚀现象初步确认）”，未直接判完整鬼域核心；选项风险 +8/+10/+14 等位于预期区间。
- 场景4（主动用鬼）：通过。独立补测成功；输出用鬼判定/压制判定语义，结果为短暂窗口与反噬，不是消灭厉鬼；死亡风险 +10，复苏风险 +25，符合高恐怖程度压制的高代价口径。
- 场景5（使用鬼烛/黄金）：通过。输出 `<sp_item_use>`；黄金隔离媒介、鬼烛抵挡袭击并急剧消耗，死亡风险受控下降（-10），没有写黄金杀死厉鬼。
- 场景6（命中杀人规律）：通过。走规律验证/直接裁定语义，没有用普通随机成功率绕过；死亡风险 +65，写成濒死与鬼烛耗尽代价；包含规律推理记录。
- 双轴A（低影响范围 + 高恐怖程度）：通过。独立补测成功；明确写“C级仅代表影响范围小”，正面压制失败，死亡风险 +25、复苏风险 +15；脚本的 `noGradeAsStrength=false` 为误报，因为原文是在批判玩家“以为 C 级等于弱”的错误判断，而不是模型采纳该口径。
- 双轴B（高影响范围 + 中等单体恐怖程度）：通过。明确写 S 级来自鬼域核心扩散规模和社会压力，少量影子鬼奴单体可被黄金隔离带阻挡；未把每个鬼奴写成源头级无解。
- 通用检查：8 个场景 raw chat 均包含 `<UpdateVariable>`、`/最近行动判定`、`<choices>` 与 `<sp_choices>`；需要规律推理的场景均包含规律推理记录；未发现写入不存在的 `/当前灵异事件/恐怖程度` 路径。
