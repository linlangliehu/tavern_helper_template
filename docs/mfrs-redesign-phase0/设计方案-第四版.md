## 0. 版本沿革

| 版次 | 核心主张 | 落地结果 |
|------|----------|----------|
| 第一～二版 | 鬼眼封案 / 神器 Logo、强动效 | 8.10.0 及更早 |
| **第三版** | 对照参考图：删眼/阵、消息内三栏（**路径 α**）、固定 host 只换皮 | **8.11.0 已发版** |
| **第四版（本文）** | 参考图 + **途尽·末日**信息架构 + **酒馆美化 2.0** 全屏挂载术 → **路径 β 全屏卷宗 HUD** | 待实现 |

### 第三版已完成（不可回退的成果）

- 无鬼眼 / 无法阵 / 无「鬼眼封案」主视觉  
- 消息内 `mfrs-msg-tri`：左档案 / 中叙事+拟办 / 右导航  
- 顶条：位置·阶段·事件·鬼域·危害  
- 拟办意见只填不发；用户楼不注入  
- aurora →「卷宗线框」token（`#3d6b66` / `#6b2a26` / `#c8c0ae`）  
- 固定 host `insertBefore` + order **10 / 20** 契约保留  
- 门禁与发布链：`12a05b5` 资源 / `26ad719` 发布 / cache `...-v8110-archive-hud`

### 第三版未解决（实机对照参考图的落差）

- 三栏仍嵌在 **ST 消息气泡**内，非全屏沉浸  
- **档案柜默认半屏**，严重挤压中栏阅读高度  
- 观感约「同构三栏的恐怖小面板」，非「全屏档案 HUD」  
- 与参考图 / 途尽卡第一眼差距仍大（布局完成度高、沉浸完成度低）

## 1. 目标与非目标

### 1.1 目标

打开 **神秘复苏模拟器 / 发布版** 后，主交互区呈现接近参考图与途尽卡的 **全屏三栏卷宗 HUD**：

```
╔══════════════════════════════════════════════════════════╗
║ 顶条：位置 · 阶段 · 事件 · 鬼域 · 危害          [退出沉浸] ║
╠════════════╦═══════════════════════════════╦═════════════╣
║ 现场档案   ║  全高叙事正文（真 #chat）      ║ 导航        ║
║ 身份 ▾     ║                               ║ · 正文      ║
║ 风险 ▾     ║  ─ 拟办意见（可折叠）─        ║ · 档案      ║
║ 事件 ▾     ║  [输入代理…………] [发送]       ║ · 关系      ║
║ 厉鬼 ▾     ║                               ║ · 柜        ║
║ 资源 ▾     ║                               ║ · 设置(灰)  ║
╚════════════╩═══════════════════════════════╩═════════════╝
```

- **中栏**占视口主高度，可舒适阅读长叙事  
- **档案柜**默认不占半屏；经「柜」展开  
- **契约零破坏**：MVU / SQL / 世界书 / 正则 33 / 脚本库序 8 / host 10·20  
- **拟办**仍只填不发；不引入生命/饱食/理智等新 MVU 字段  

### 1.2 非目标

| 不做 | 原因 |
|------|------|
| 整站替换 SillyTavern 皮肤（美化 2.0 模块 B） | 越界、与卡过滤冲突 |
| 整包移植途尽 `[前端]正文` 13 万字正则 HTML | 维护成本、领域字段不兼容 |
| 整包移植美化 2.0 全文（悬浮球+通用关键词拆分） | 无卡过滤、数据模型错误 |
| 新增生命/饱食/理智 MVU | 第三版起契约禁止 |
| 路径 α 推倒重写数据层 | α 的 HTML builder / 字段读取可复用 |
| 手改发布版 PNG 作主路径 | 仍走 `publish-card` |

## 4. 布局规格

### 4.1 桌面（≥1100px）

| 区域 | 规格 |
|------|------|
| 壳 | `#mfrs-hud-shell`：`position:fixed; inset:0; z-index:10000`；`display:grid` |
| 网格 | 行：`auto 1fr`（顶条 + 主体）；列：`minmax(240px,280px) minmax(0,1fr) minmax(180px,220px)` |
| 顶条高 | 40–48px；虚线底边 + 情境 chips |
| 左栏 | 可滚动；标题「现场档案」；`<details>` 五段默认策略见 §5 |
| 中栏 | flex 列：`#chat` 宿主 `flex:1` + 拟办折叠 + 输入代理 sticky |
| 右栏 | 竖向导航按钮列表；当前项高亮（冷青实底/描边） |
| 线框 | 直角；尸青主边 + 血赤弱 outline；四角 L 标（弱） |
| Token | `--mfrs-corpse-cyan:#3d6b66` · `--mfrs-blood-red:#6b2a26` · `--mfrs-bone-white:#c8c0ae` · ink `#0a0b0b` |

### 4.2 窄屏

| 宽度 | 行为 |
|------|------|
| ≤1100 | 左 220 / 右 160 |
| ≤800 | 中栏全宽；左/右改为抽屉（导航按钮打开） |
| 触控 | 主要按钮 ≥44px；`prefers-reduced-motion: reduce` 关动画 |

### 4.3 z-index

| 层 | z-index |
|----|---------|
| 全屏壳 | ~10000 |
| 壳内抽屉/柜覆盖 | 10010–10050 |
| ACU 对话框等 | 保持其超高 z（2e9 级），须压在壳之上可点 |

### 4.4 与 ST 宿主关系

- **不删除** `#top-bar` / `#send_form` 节点；由壳盖住  
- **直播移动** `#chat` 入中栏宿主；退出时按 parent/nextSibling **精确还原**  
- 输入：壳内 textarea → `#send_textarea` + `input` +（发送时）`#send_but`  
- 拟办：只写 textarea，**不**点发送

## 5. 信息架构与字段映射

### 5.1 顶条（情境条）

| UI 标签 | 路径 | 缺省 |
|---------|------|------|
| 位置 | `所在位置` | 未知 |
| 阶段 | `主线进度.当前阶段` | 开局接入 |
| 事件 | `当前灵异事件.事件代号` | 未立案灵异事件 |
| 鬼域 | `当前灵异事件.鬼域状态` | 未确认 |
| 危害 | `当前灵异事件.危害等级` | 未知 |

**不映射**：途尽的时间、天气（无 schema，禁止造字段）。

### 5.2 左栏「现场档案」

| 折叠段 | 行项目 | 路径 | 备注 |
|--------|--------|------|------|
| **身份** | 姓名 | `姓名` | |
| | 性别 | `性别` | |
| | 身份 | `身份` | 对位途尽「蜕变层级」 |
| | 年龄 | `初始年龄` | 可选 |
| | 能力 | `特殊能力描述` | |
| | 代价 | `消耗代价` | |
| **风险** | 死亡风险 | `风险值` | 条+等级；**替代**途尽生命 |
| | 复苏风险 | `驭鬼者状态.总复苏风险` → 回退 `厉鬼复苏程度` | **替代**途尽理智 |
| | 状态 | `状态` | 文本；**替代**途尽名声条 |
| **事件** | 事件代号/危害/地点/鬼域/处理 | `当前灵异事件.*` | |
| **厉鬼** | 列表 | 优先 `驭鬼者状态.已驾驭厉鬼[]`；兼容 `驾驭厉鬼[]` | 代号/恐怖/复苏/死机等 |
| **资源** | 拼图/物品/黄金 | `灵异资源.*`；物品兼容顶层 `灵异物品[]` | |

**明确不做左栏**：生命/饱食/理智三维、资质八维、种族、时间天气。

### 5.3 右栏导航

| 键 | 行为 |
|----|------|
| 正文 | 中栏显示叙事；滚到底/聚焦 chat |
| 档案 | 强调左栏；或中栏展示 `收录档案[]` / `收录规律[]`（P1） |
| 关系 | 中栏：`在场人物[]`（解析 `名-描述`） |
| 柜 | 展开固定 host / 档案柜（C1） |
| 设置 | 默认 disabled 或简易：自动全屏、字号（P2） |

**不抄途尽**：势力独立页、图鉴四页、探索、商城、通讯、兽宠（势力关系若需可并入关系弱展示 `势力关系.*`，非 P0）。

### 5.4 中栏

| 区块 | 数据 / 行为 |
|------|-------------|
| 叙事 | 真 `#chat`（流式安全） |
| 拟办 | `行动建议[]`：`选项/思路/主要风险/预期收益` → 按钮只填输入框 |
| 输入代理 | 同步 ST 发送链路 |
| 开局表单 | 若仍在首楼 HTML：全屏时允许中栏滚动查看；不二次实现途尽开局选择器 |

### 5.5 途尽 → MFRS 速查

```
途尽·生命条     →  风险值（死亡风险）
途尽·理智条     →  驭鬼者状态.总复苏风险
途尽·饱食条     →  ∅
途尽·姓名/性别  →  姓名/性别
途尽·蜕变层级   →  身份
途尽·坐标       →  所在位置
途尽·时间/天气  →  ∅
途尽·交际       →  关系 ← 在场人物
途尽·正文导航   →  正文
途尽·设置       →  设置（可灰）
（途尽无）      →  柜 / 拟办 / 事件档案 / 驾驭厉鬼
```

完整表见对话交付的《途尽布局规格 → 神秘复苏字段映射表》；实现以 **本文 §5 + schema.ts** 为准。

## 6. 技术架构

### 6.1 模块落点（推荐）

**不新增脚本库项**（保持 8 脚本序契约），在现有 **消息内面板** bundle 内增加全屏模块：

```
src/神秘复苏模拟器/脚本/消息内面板/
  index.ts              # 入口：卡过滤、事件、cleanup
  （逻辑分区，可同文件或拆分）
  - α 消息内 tri（保留，全屏 ON 时 hide/dismantle）
  - β fullscreen-hud    # 壳 DOM/CSS、reparent、代理、导航
```

可选后续：若体积过大再拆文件，仍由同一 webpack 入口打包。

### 6.2 生命周期

```
卡激活 ──► ensureShell() ──► (D1? mount : 显示入口按钮)
mount:
  1. 保存 #chat parent/nextSibling
  2. body.overflow=hidden；壳 .active
  3. centerHost.appendChild(#chat)
  4. 渲染左/顶/右；绑定输入代理与导航
  5. hide α tri 与默认半屏柜视觉
unmount / 切非神秘复苏卡 / cleanup:
  1. 还原 #chat
  2. 去 .active；恢复 overflow
  3. 恢复 α 与 host 可见性策略
```

### 6.3 与现有组件契约

| 组件 | 全屏 ON | 全屏 OFF |
|------|---------|----------|
| α `mfrs-msg-tri` | 隐藏/拆除重复侧栏 | 8.11 行为 |
| 固定状态栏 host | 不删节点；视觉隐藏或归「柜」 | insertBefore + 10/20 |
| 界面美化 | token 继续生效 | 同 |
| hotfix / MVU / DB | 不动 | 不动 |
| ACU 弹窗 | 须仍可点（z 更高） | 同 |

### 6.4 数据读取

- 与 α 相同：`readStatusForMessage(latestAiMes)` / `getVariables({type:'message',...}).stat_data`  
- 刷新：消息渲染结束、MVU 更新、CHAT_CHANGED；**禁止** 2.5s 强制滚底（美化 2.0 反模式）  
- **只读**展示；拟办不写 MVU；柜内 CRUD 仍走原 ACU API  

### 6.5 明确不采用的途尽技术

- 用正则把 `<content>` 换成整页 HTML 应用  
- 在消息 UI 内大量 `Mvu.replaceMvuData` 改背包类状态  
- 依赖 `aspect-ratio` 假全屏代替 `position:fixed` 真全屏
