**设计要点：**

1. **货币系统：**
   - 货币名称：调查点（Investigation Points）
   - 获取途径：消息 +1、线索 +5、事件 +10、对抗厉鬼 +15
   - 消耗：单抽 10 点，十连 90 点（9 折优惠）

2. **保底机制：**
   - 十连保底：必出 ★★★
   - 50 抽保底：必出 ★★★★
   - 100 抽保底：必出 ★★★★★★

3. **物品池（19 种灵异物品，符合神秘复苏原著设定）：**

   **★★★★★★（神话，0.5%）：**
   - 源头碎片：可改写厉鬼杀人规律（不能让厉鬼死机）

   **★★★★★（传说，2%）：**
   - 鬼域：可关押厉鬼
   - 鬼差制服：可抵御厉鬼袭击

   **★★★★（史诗，5%）：**
   - 黄金手掌：可击退厉鬼
   - 饿死鬼的香烟：可暂时压制厉鬼
   - 鬼邮件：可传递信息
   - 鬼奴隶：可使役厉鬼

   **★★★（稀有，15%）：**
   - 红色鬼烛：照亮鬼域，驱散黑暗
   - 鬼钱：购买灵异物品或服务
   - 卫星定位手机：定位厉鬼或驭鬼者
   - 压制类物品：压制特定厉鬼

   **★★（普通，30%）：**
   - 基础灵异物品（5 种）

   **★（常见，47.5%）：**
   - 入门级灵异物品（3 种）

4. **线索/知识物品：**
   - 线索（提升档案进度）：普通 5%、重要 10%、核心 25%、决定性 50%
   - 知识（提升规律进度）：基础 5%、深入 10%、核心 25%、禁忌 50%

5. **四个物品池：**
   - 全物品池（灵异物品 + 线索 + 知识，均匀分布）
   - 厉鬼档案池（线索权重提升 2 倍）
   - 厉鬼规律池（知识权重提升 2 倍）
   - 灵异物品池（仅灵异物品）

**注意事项：**

1. **原著设定约束（来自用户 8 轮反馈）：**
   - ❌ 厉鬼不存在"伤害"概念，只能击退/死机/关押
   - ❌ 灵异物品不是厉鬼本身（不能是鬼画、鬼寿衣）
   - ❌ 灵异物品无负面效果
   - ❌ 不包含普通物品（盐、符纸、普通蜡烛）
   - ❌ 不包含势力科技（总部电话、卫星系统）
   - ❌ 不包含唯一性物品（人皮纸等原著特定物品）
   - ❌ 不包含未沾染灵异能力的黄金物品
   - ✅ 所有物品必须"沾染了灵异，拥有灵异能力"

2. **实现优先级：**
   - 先实现核心抽卡逻辑，验证概率分布正确
   - 再实现 UI 界面，确保视觉风格统一
   - 最后实现数据同步，确保与数据库前端无缝集成

3. **代码插入位置：**

## 技术债务

### 1. dist/ 产物本地残留
- **问题：** 本地 `npm run build` 会生成 dist/，但不应提交
- **解决方案：** .gitignore 已配置，需确保 git status 不显示 dist/
- **优先级：** 低

### 2. CDN 缓存刷新延迟
- **问题：** jsdelivr GitHub CDN 缓存最多 24 小时
- **影响：** 真机测试可能加载旧版本代码
- **解决方案：** 使用 `?t=timestamp` 破缓存，或等待 24 小时
- **优先级：** 低（不影响最终发布）

### 3. 部分表 row_id 空字符串（已修复）
- **问题：** sheet_clues、sheet_chronicle 等表 row_id 为空字符串
- **根因：** vendor fallback 逻辑未检查 content 长度
- **解决方案：** v6.29 已修复
- **状态：** ✅ 已解决
