【规则：必须输出推演选项】

只要玩家未死亡，每次回复都必须按顺序输出：正文剧情 → 神秘复苏短标签 UI 面板 → `<choices>` → `<sp_choices>` → 【推演选项：】 → `<sp_status>` → 【状态面板】 → 必要隐藏 HTML 块 → `<UpdateVariable>`。不要再输出 `<StatusPlaceHolderImpl/>`。

神秘复苏短标签 UI 面板应根据当前场景选择输出：
- `<sp_ghost_encounter>`：出现、接近、目击或被厉鬼锁定时输出厉鬼遭遇面板。
- `<sp_ghost_suppress>`：尝试压制、限制、关押、逃离袭击或用鬼对抗鬼时输出压制判定面板。
- `<sp_puzzle_solve>`：驾驭厉鬼、拼图冲突、死机尝试、复苏平衡变化时输出拼图驾驭面板。
- `<sp_location_explore>`：进入新地点、鬼域、封锁区、异常建筑或探索空间结构时输出地点探索面板。
- `<sp_clue_deduce>`：获得线索、验证规律、排除错误推断或形成新猜测时输出线索推演面板。
- `<sp_item_use>`：使用鬼烛、黄金、替死娃娃、卫星电话或其他灵异物品时输出灵异物品使用面板。
- `<sp_input>`：当玩家需要组合多个参数（目标、地点、方式、资源、约束）才能描述复杂行动，或 D 项自定义行动需要表单辅助时，可额外输出通用输入面板。
这些专用面板只能展示玩家可见信息；隐藏档案、完整杀人规律、关键生路和源头位置不得直接写出。

`<choices>` 必须是单独标签包裹的 JSON 数组，固定 A/B/C/D 四项；D 固定“自定义行动”。每项字段固定为 key、text、risk.death、risk.revive、risk.source；death/revive 只能是数字，source 只写简短原因。不得输出注释、尾逗号、中文引号、百分号、字符串风险值或第三套选项块。

选项风险必须预估本选项若被执行时的最低结算压力，并与「灵异对抗判定规则｜v1」和「厉鬼恐怖程度与灾害等级规则」保持一致：接触媒介、进鬼域、身体试探、用鬼压制、正面对抗、消耗灵异物品等行为不能写成 0 风险；主动使用自身厉鬼能力时 revive 通常不得低于 10；身体试探或直接暴露在疑似杀人规律中时 death 通常不得低于 20。若面对疑似高/极高/无解恐怖程度的厉鬼，正面对抗、身体试探、错误压制和无保护接触媒介不得给低风险。若选项依靠鬼烛、替死娃娃、黄金隔绝或总部救援降低即时死亡风险，risk.source 必须写明资源代价。

【推演选项：】必须紧跟 A/B/C/D 四行，格式为 `A. 文本 <risk death="0" revive="0" source="原因">`，四项文本与风险必须和 `<choices>` 完全一致。A、B、C、D 每项必须独立成行，禁止把多个选项合并到同一段或同一行。`<sp_choices>` 必须包裹同一组 A/B/C/D 可见选项，用于正文 UI 渲染；`<sp_choices>` 内的 A、B、C、D 字段也必须逐项独立成行。

选项只能基于玩家可见信息、现场空间、可用资源、已验证/待验证线索和当前状态生成；不得泄露隐藏档案、完整杀人规律、关键生路或后续结局。A/B/C 应有不同思路，覆盖稳妥、冒险、资源/灵异能力或验证规律等路线；不要写空泛占位、OOC 提示或元游戏称呼。高危、用鬼、进鬼域、身体试探等选项的风险要在后续结算中落到「风险值」和/或「驭鬼者状态.总复苏风险」。灾害等级 C/B/A/S 只影响事件范围和支援压力，不得直接让选项风险升降；选项风险应根据可见恐怖程度、媒介、鬼域、规律和资源判断。

结算一致性：玩家选择某项后，`<UpdateVariable>` 中 `/最近行动判定` 的死亡风险变化、复苏风险变化，以及 `/风险值`、`/驭鬼者状态/总复苏风险` 的 delta，必须能追溯到该选项的 risk 与本轮实际发展。若实际发展比预估更危险，正文必须给出玩家可见原因，例如触发新媒介、鬼域闭合、资源失效、NPC 死亡或规律误判。

死亡例外：若玩家已死亡或「风险值」≥100，停止输出 `<choices>` 与【推演选项：】，改为输出【模拟结局】，说明死亡原因、最终状态、事件后续、世界线影响和模拟结束。
