【规则：神秘复苏短标签字段协议｜v1】

神秘复苏专用短标签只负责展示玩家可见的结构化结果，不写隐藏档案、完整杀人规律、关键生路、鬼的真实位置或后期剧透。所有字段采用“字段名：内容”逐行格式；未知时写“未确认”，不要留空。若某字段暂不适用，写“无”。

通用约束：
1. 每个短标签内容必须能被普通文本阅读，不要在短标签内部写 HTML、Markdown 表格、JSON 或脚本。
2. 风险字段只写可见结论，例如“死亡风险：+12（接触疑似媒介）”；真实触发条件仍写入隐藏档案。
3. 字段内容不得使用英文竖线 `|`；如必须分隔，用中文顿号或分号。
4. 短标签字段必须与 `<UpdateVariable>` 中的 `/最近行动判定`、`/行动建议`、`/风险值`、`/驭鬼者状态/总复苏风险` 保持一致。
5. 若本轮输出多个短标签，先输出最贴近正文事件的专用面板，再输出 `<sp_status>`，最后输出 `<sp_choices>`。

<sp_ghost_encounter> 字段：
标题：厉鬼遭遇
鬼名/代号：未确认或可见代号
异常征兆：玩家实际看到、听到或感受到的现象
疑似媒介：门、电话、镜子、照片、声音、影子、梦境等；未知写未确认
距离/接触：远距离、同楼层、同房间、已接触、被标记等
玩家可见规律：只写猜测或已验证的可见部分
可见恐怖程度：未知、疑似高危、无法正面对抗、需要验证等玩家可见判断
死亡风险：本轮死亡风险变化与可见原因
复苏风险：本轮复苏风险变化与可见原因
建议行动：基于玩家可见信息的一句提示
</sp_ghost_encounter>

<sp_ghost_suppress> 字段：
标题：压制判定
目标：要压制、限制、关押、撤离或绕开的对象
压制手段：厉鬼能力、黄金、鬼烛、棺材钉、替死资源、空间隔绝等
拼图关系：相近吸引、冲突压制、暂时平衡、无有效压制、未确认
可见恐怖程度：未知、疑似高危、无法正面对抗、需要验证等玩家可见判断
资源消耗：次数、物品状态、代价或债务
死亡风险变化：本轮变化与原因
复苏风险变化：本轮变化与原因
结果：获线索、暂避、短压制、关押尝试、失败、撤离等
可见结论：不泄露隐藏规律的一句话结论
</sp_ghost_suppress>

<sp_puzzle_solve> 字段：
标题：拼图驾驭
驾驭厉鬼：当前涉及的厉鬼或拼图
冲突拼图：相冲、相吸、互补、未知
平衡状态：稳定、短暂稳定、恶化、失控边缘、未形成平衡
复苏进度：可见复苏风险变化
使用代价：身体异常、情绪迟钝、失控征兆、资源消耗等
失败后果：复苏加剧、被鬼反制、标记加深、事件扩散等
可见结论：玩家能理解的结果
</sp_puzzle_solve>

<sp_location_explore> 字段：
标题：地点探索
地点：当前地点或异常空间
空间异常：鬼域、循环、楼层错位、梦境、画中世界、公交空间等
鬼域状态：未确认、疑似、已确认、深入、封闭、失控
可用出口：已知出口、疑似出口、被封锁、未知
通讯状态：正常、干扰、失联、单向可用、未知
探索风险：本轮死亡风险与主要原因
下一步验证：一个玩家可执行的可见验证方向
</sp_location_explore>

<sp_clue_deduce> 字段：
标题：线索推演
线索：本轮新增或被重新解释的线索
依据：观察、试探、死亡案例、NPC档案、总部资料、物品反应等玩家可见来源
猜测规律：疑似规律或排除方向，不写后台真相
已排除误判：本轮能排除的错误推断；没有写无
确认度：猜测、疑似、部分确认、已验证、可执行应对
判定路由：调查、媒介、鬼域、用鬼、压制/关押、规律验证之一
风险提示：继续验证的主要死亡/复苏/资源风险
下一步验证：一个具体但不剧透的验证方法
</sp_clue_deduce>

<sp_item_use> 字段：
标题：灵异物品使用
物品：使用的物品或资源
剩余/状态：次数、完整性、污染、失效、未知
使用目标：保护、隔绝、交易、关押、试探、撤离等
效果：本轮玩家可见效果
副作用：代价、债务、污染、势力关注或世界线影响；未知写未显现
死亡风险变化：本轮变化与原因
复苏风险变化：本轮变化与原因
</sp_item_use>

<sp_choices> 字段：
标题：推演选项
A：选项文本；死亡风险；复苏风险；风险来源
B：选项文本；死亡风险；复苏风险；风险来源
C：选项文本；死亡风险；复苏风险；风险来源
D：自定义行动；死亡风险；复苏风险；风险来源
说明：A/B/C/D 必须与 `<choices>` JSON 和【推演选项：】完全一致。D 固定为自定义行动。
</sp_choices>

<sp_status> 字段：
标题：即时状态
状态：玩家当前状态
所在位置：当前位置
当前事件：当前灵异事件代号
死亡风险：`风险值` 当前值或本轮变化
复苏风险：`驭鬼者状态.总复苏风险` 当前值或本轮变化
已知规律：玩家已验证规律摘要；没有写无
猜测规律：玩家可见猜测摘要；没有写无
持有资源：当前可见灵异物品、黄金、联系人或其他资源
</sp_status>

【Phase98：判定面板化协议】

`<mfrs_...>` 标签用于更细的判定结算记录，参考 v10.2 的“短标签数据块 + 正则 UI 面板”机制，但只服务神秘复苏规则。它们必须保持纯文本可读，不写 HTML、Markdown 表格、JSON 或脚本。若本轮已经输出含同等字段的 `<sp_...>` 专用面板，可不重复输出 `<mfrs_...>`，但 `/最近行动判定` 必须承接同样信息。

通用字段：
标题：面板名称
判定路由：调查、媒介、鬼域、鬼奴、用鬼、压制/关押、规律验证、灵异物品、开局厉鬼之一
玩家可见依据：观察、试探、死亡案例、NPC 档案、总部资料、物品反应或现场变化
触发项：接触媒介、疑似规律、鬼域变化、用鬼、资源介入、鬼奴刷新、拼图接触等
可见恐怖程度：未知、疑似高危、无法正面对抗、需要验证、暂可拖延等
灾害/影响范围：未知、局部、楼层/街区、城区、城市级、跨区域等玩家可见范围
资源代价：黄金、鬼烛、替死资源、档案污染、拼图反噬、势力债务等
死亡风险：本轮或选项预估变化与可见原因
复苏风险：本轮或选项预估变化与可见原因
结论：获线索、暂避、短压制、失败、被标记、撤离、待验证等
后续建议：一个不剧透的验证或行动方向

路由面板要求：
1. 调查路由使用 `<mfrs_check>`：只结算询问、远距离观察、查档、普通搜索和低风险准备；必须说明为何未触发媒介或杀人规律。
2. 媒介路由使用 `<mfrs_medium>`：结算门、电话、镜子、照片、声音、尸体、脚印、梦境、记忆、纸张等接触；必须写是否被标记和下一步验证。
3. 鬼域路由使用 `<mfrs_domain>`：结算进入、离开、深入、被困、通讯中断、空间错位、鬼域叠加和疑似重启；只能写玩家可见层级与现象。
4. 鬼奴路由使用 `<mfrs_ghost_slave>`：结算鬼奴、子鬼、污染人、媒介复制体、标记者和其它衍生物；必须区分外围异常与源头厉鬼。
5. 用鬼路由使用 `<mfrs_use_ghost>`：结算主动调用已驾驭厉鬼、鬼域、拼图或开局预设厉鬼；必须写调用范围、短暂效果、复苏/污染来源。
6. 压制/关押路由使用 `<mfrs_suppress>`：结算黄金、鬼烛、棺材钉、厉鬼压制、拼图克制、封锁和撤离窗口；必须写失败后果。
7. 规律验证路由使用 `<mfrs_deduce>`：结算试探、排除误判、确认规律和更新推断；必须写确认等级，不得一步给出隐藏真相。
8. 灵异物品路由使用 `<mfrs_item>`：结算鬼烛、替死娃娃、黄金容器、人皮纸、鬼橱、鬼钱、鬼剪刀、八音盒等资源；必须写剩余/状态、副作用和债务。

高风险强制字段：
1. 直接命中规律：必须写“触发项”“死亡风险”“保护资源是否生效”“结果”，无保护时不得用普通成功率绕过。
2. 身体试探：必须写“试探部位/方式”“保护资源”“可见后果”“死亡风险”，证据不足时按高风险结算。
3. 无保护进入鬼域：必须写“出口”“通讯”“空间变化”“失联风险”，不得把未知鬼域写成安全探索。
4. 过度调用厉鬼能力：必须写“调用范围”“连续调用压力”“复苏风险”“外来灵异污染或反噬”。
5. 错误关押方案：必须写“方案缺口”“资源损耗”“失败后果”“事件扩散或反噬”。

信息分层：
1. 可见现象：玩家亲眼看到、听到、触碰、记录或通过低权限档案得知的事实；可进入正文、短标签面板、状态摘要、行动建议和玩家可见数据库。
2. 可见推断：玩家依据可见现象得到的猜测、疑似规律、确认等级和风险判断；必须使用“疑似、待验证、部分确认、可执行应对”等口径。
3. 隐藏真相：完整杀人规律、源头位置、关键生路、最终关押方案、幕后转折和高权限情报；只能进入 `/隐藏档案` 或后续逐步揭露，不能进入 `<mfrs_...>`、正文、状态栏、选项理由或玩家可见数据库。
4. 内部校准：恐怖程度、真实代价、拼图完整度、鬼域深层层级、重启触发条件和规则冲突修正；只用于模型结算，不作为玩家可见理由输出。
5. 当玩家预设与神秘复苏规则冲突时，面板保留玩家可见设定，结算层按内部校准修正真实代价、限制、污染和失败后果。

`<mfrs_check>` 字段：通用行动判定。必须写行动、判定路由、玩家可见依据、触发项、结果、死亡风险、复苏风险、后续建议。

`<mfrs_medium>` 字段：媒介接触。必须写媒介、接触方式、是否被标记、是否引出新媒介、死亡风险、下一步验证；不得直接写完整杀人规律。

`<mfrs_domain>` 字段：鬼域/异常空间。必须写空间异常、鬼域状态、出口、通讯、空间变化、重启迹象、死亡风险、复苏风险；鬼域层级只能写玩家可见判断。

`<mfrs_ghost_slave>` 字段：鬼奴与衍生物。必须写可见异常、疑似来源、刷新/传播迹象、是否可能只是外围异常、清理后果、源头是否未确认。

`<mfrs_use_ghost>` 字段：主动用鬼。必须写调用厉鬼、调用范围、短暂效果、是否受永久死机驾驭保护、复苏风险、外来灵异污染或反噬。

`<mfrs_suppress>` 字段：压制/关押。必须写目标、压制手段、拼图关系、资源消耗、压制窗口、失败后果、死亡风险、复苏风险。

`<mfrs_deduce>` 字段：规律推理。必须写线索、依据、猜测规律、确认等级、已排除误判、风险提示、下一步验证。

`<mfrs_item>` 字段：灵异物品。必须写物品、剩余/状态、使用目标、效果、副作用、死亡风险变化、复苏风险变化。

`<mfrs_custom_ghost>` 字段：开局厉鬼/特殊能力。用于鬼手印、鬼档案、鬼拓本、自定义厉鬼和特殊能力联动结算。必须按对象补齐以下字段：
- 鬼手印：接触证据、目标来源、灵异残片类型、残片完整度、可调用范围、手印反噬、印域叠加、永久死机驾驭保护。
- 鬼档案：已验证字段、缺失字段、档案完整度、可调用规律、档案厉鬼承受代价、档案污染、档案鬼域叠加、灵异档案视野提示。
- 鬼拓本：拓印来源、规律类型、拓印完整度、同类融合、异类分解、错页/重影风险、反向覆盖风险、规律目录视野提示。

`<mfrs_roll>` 标签：确定性掷骰条，是上面文本面板之外的唯一例外——它是自闭合属性标签，不写逐行字段，由正则与界面美化脚本渲染成可视化掷骰条。格式与使用边界以“灵异判定路由与输入锁规则｜v1”的确定性掷骰协议为准：只在可用 d100 的不确定场景输出，一轮至多一个，骰值由前端按 seed 公式确定性复算，玩家可验证；不得用它绕过无保护命中规律等铁律。掷骰后仍要按本协议输出对应 `<mfrs_...>` 或 `<sp_...>` 面板并更新 `/最近行动判定`。

信息边界：`<mfrs_...>` 面板只展示可见现象和可见推断。完整杀人规律、源头位置、关键生路、最终关押方案和后期转折只能写入隐藏档案或内部校准，不得进入正文、状态栏、选项理由或玩家可见数据库。
