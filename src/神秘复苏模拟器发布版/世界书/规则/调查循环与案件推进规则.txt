【规则：调查循环与案件推进｜v1】

神秘复苏模拟器的核心不是升级打怪，而是灵异案件的调查、误判、代价和收束。每个正在进行的灵异事件，都必须围绕以下循环推进：

遭遇异常 → 收集线索 → 试探规律 → 排除误判 → 使用资源/鬼能力 → 临时压制/关押/撤离/死亡。

一、循环阶段定义

1. 遭遇异常：玩家首次发现异常现象、尸体、媒介、空间错位、失踪、声音、影子、梦境、照片、门、电话等。此阶段只给玩家可感知事实，不直接给完整规律。
2. 收集线索：玩家通过观察、询问、查档案、查看监控、搜索现场、联系总部或对比死亡案例获得信息。必须区分已观察事实、传闻和推断。
3. 试探规律：玩家主动做低强度或受控测试，例如远距离观察、隔绝媒介、用物品替代身体、让 NPC 或设备验证。试探要产生明确结果或代价。
4. 排除误判：当新证据推翻旧判断时，必须保留错误推断，不要静默删除；误判会影响后续选项风险、NPC 决策或资源消耗。
5. 使用资源/鬼能力：玩家用鬼烛、黄金、替死娃娃、人皮纸、总部支援、自身厉鬼能力或其他资源换取行动窗口。必须记录消耗、副作用、复苏压力或势力债务。
6. 临时压制/关押/撤离/死亡：事件阶段性收束。成功只能是短压制、关押、封锁、转移、获知关键线索或活着离开；不能写成彻底消灭厉鬼。

二、每轮至少推进一个维度

当玩家处于灵异事件中，每次回复至少推进以下一项；如果没有推进，必须说明玩家主动停滞或外部时间压力正在增加。

1. 线索增加：新增观察、档案、死亡案例、现场反应、物品变化或 NPC 证词。
2. 风险增加或降低：根据行动让 `风险值`、复苏风险、事件扩散或资源状态变化。
3. 资源消耗：鬼烛、黄金、替死娃娃、通讯、车辆、联系人、总部支援、灵异物品次数等发生变化。
4. 规律确认：猜测升级为疑似、部分确认、已验证或可执行应对。
5. 误判记录：发现错误推断、假线索、巧合、生路陷阱或 NPC 错误判断。
6. 地点状态变化：鬼域展开、出口消失、通讯中断、封锁建立、楼层错位、异常空间加深。
7. NPC 状态变化：伤亡、失踪、被标记、撤离、背叛、提供线索、被总部接管。
8. 世界/势力压力变化：总部关注、社会公开度、势力介入、世界线偏移。

三、规律确认度

规律只能逐级确认，不得一步到位泄露完整真相。

1. 猜测：只有单一现象或玩家直觉；写入 `当前灵异事件.猜测杀人规律` 或 `规律推理记录`。
2. 疑似：有两类以上可见证据支持，但缺少代价验证；仍不能转入已知规律。
3. 部分确认：某个条件已被验证，例如声音、视线、距离、回应、照片、梦境等一部分成立；可写入已知规律的片段，但不得补全隐藏部分。
4. 已验证：经过观察、试探、死亡案例、总部档案或可靠实验确认；可转入 `当前灵异事件.已知杀人规律`。
5. 可执行应对：玩家不仅知道触发条件，还知道可行规避或压制方法；必须来自已验证信息、资源和现场条件，不能凭空出现。

四、误判保留

1. 错误推断必须保留在 `当前灵异事件.错误推断` 或 `规律推理记录`，不要因为新线索出现就删除旧误判。
2. 玩家重复错误推断时，应提高相关选项的 death/revive 风险，或让 NPC 对玩家判断产生怀疑。
3. NPC 的错误判断也可以记录为可见误导线索；若涉及幕后真相或关键生路，真实部分写入 `隐藏档案.误导线索`。
4. 当误判被排除时，在【本轮摘要】的“线索”行写“已排除误判：...”，并在 `/最近行动判定` 写明可见依据。

五、事件扩散规则

以下情况会提高 `当前灵异事件.扩散趋势`、`已死亡人数`、`主线进度.世界压力.总部关注度` 或 `社会公开度`：

1. 玩家拖延处理，导致媒介继续传播、鬼域扩大、尸体增多或封锁失败。
2. 玩家错误处理，例如破坏媒介却释放新媒介、公开异常、误导群众、把普通人带入危险区。
3. 玩家使用厉鬼能力或灵异物品造成明显公开影响。
4. NPC 伤亡、失踪、被标记，或关键人物/资源归属改变。
5. 总部、朋友圈、灵异论坛、国王组织、民间驭鬼者、鬼邮局等势力介入。

扩散更新必须小步且可解释，不要无依据把事件直接提升为 S 级或全国公开。

六、行动建议生成

A/B/C/D 选项必须根据当前循环阶段生成，避免每回合重复“逃跑、硬刚、调查”。

1. 遭遇异常阶段：选项应覆盖远离媒介、观察异常、求援、保护自己或他人。
2. 收集线索阶段：选项应覆盖查档案、问询、现场取证、对比死亡案例、联系总部。
3. 试探规律阶段：选项应覆盖低风险试探、资源保护下试探、放弃试探撤离、D 自定义行动。
4. 排除误判阶段：选项应覆盖验证新证据、修正计划、警告 NPC、处理误导线索。
5. 使用资源/鬼能力阶段：选项应明确资源代价和复苏风险，不得写成免费按钮。
6. 收束阶段：选项应覆盖封锁、关押、撤离、等待总部接管、处理后续世界线影响。

七、输出与变量

1. 正文写玩家可见推进结果；【本轮摘要】只写当前循环阶段最关键的线索、风险、资源或下一步方向。
2. `<UpdateVariable>` 必须记录本轮推进维度：至少更新最近行动判定；如有线索、误判、规律、风险、资源、地点、NPC 或势力变化，也必须更新对应路径。
3. 若本轮只是等待，也要推进时间压力：扩散趋势、总部关注、风险或下一步推进提示至少一项发生变化。
