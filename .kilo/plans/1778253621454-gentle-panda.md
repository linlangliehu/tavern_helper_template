# 神秘复苏模拟器实时修改配置计划

## 问题诊断

`tavern_sync.yaml` 中配置了两个角色卡：
1. **角色卡示例** — 指向 `示例/角色卡示例/index`，酒馆中名称为 `呕吐内心的少女`
2. **神秘复苏模拟器** — 指向 `src/神秘复苏模拟器/index`，酒馆中名称为 `神秘复苏模拟器`

### 冲突分析

**会产生问题**。当 `pnpm watch` 运行时，它自动执行 `node tavern_sync.mjs watch 所有 -f`，即**同时监听所有配置**。如果 `呕吐内心的少女` 这个角色卡不在你的酒馆中，tavern_sync 连接酒馆后会尝试查找它但找不到，可能导致报错或影响 `神秘复苏模拟器` 的正常同步。

### 其他需要修改的地方

| # | 文件 | 问题 | 修改方式 |
|---|------|------|----------|
| 1 | `tavern_sync.yaml` | `角色卡示例` 配置指向不存在的酒馆角色卡，会在 watch 时报错 | 删除或注释掉 `角色卡示例` 配置，只保留 `神秘复苏模拟器` |
| 2 | `tavern_sync.yaml` | `user名称: 青空黎` 不是你的用户名 | 改为你自己在酒馆中的用户名，否则提示词中的名字不会被替换为 `<user>` 宏 |
| 3 | `src/神秘复苏模拟器/index.yaml` 第215行 | `[界面]状态栏` 正则的 URL 是 CDN 远程链接 `https://testingcf.jsdelivr.net/...`，无法实时更新 | 改为本地 Live Server 地址 `http://localhost:5500/dist/神秘复苏模拟器/界面/状态栏/index.html`（发布时改回 CDN 或用 `pnpm build` 后的内联结果） |
| 4 | `src/神秘复苏模拟器/index.yaml` 第456行 | MVU 脚本使用 CDN import，实时编写时改动本地代码不会影响这个 CDN 引用 | 实时编写阶段不需要改（MVU 是第三方库，不需要本地修改） |
| 5 | `src/神秘复苏模拟器/index.yaml` 第462行 | 变量结构脚本使用 CDN import `https://testingcf.jsdelivr.net/gh/StageDog/tavern_helper_template/dist/神秘复苏模拟器/脚本/变量结构/index.js` | 实时编写时需改为本地 Live Server 地址 `http://localhost:5500/dist/神秘复苏模拟器/脚本/变量结构/index.js` |

## 修改计划

### 步骤 1：修改 `tavern_sync.yaml`

- 删除 `角色卡示例` 配置块（第9-27行）
- 将 `user名称` 改为你自己的用户名

修改后内容：
```yaml
user名称: 扮演者

配置:
  神秘复苏模拟器:
    类型: 角色卡
    酒馆中的名称: 神秘复苏模拟器
    本地文件路径: src/神秘复苏模拟器/index
    导出文件路径: src/神秘复苏模拟器/神秘复苏模拟器
```

### 步骤 2：修改 `src/神秘复苏模拟器/index.yaml` 中 `[界面]状态栏` 正则

将第215行的 CDN 链接改为本地 Live Server 链接：

```
# 修改前
$('body').load('https://testingcf.jsdelivr.net/gh/StageDog/tavern_helper_template/dist/神秘复苏模拟器/界面/状态栏/index.html')

# 修改后
$('body').load('http://localhost:5500/dist/神秘复苏模拟器/界面/状态栏/index.html')
```

### 步骤 3：修改 `src/神秘复苏模拟器/index.yaml` 中变量结构脚本的 import

将第462-463行的 CDN import 改为本地 Live Server 地址：

```
# 修改前
import 'https://testingcf.jsdelivr.net/gh/StageDog/tavern_helper_template/dist/神秘复苏模拟器/脚本/变量结构/index.js';

# 修改后
import 'http://localhost:5500/dist/神秘复苏模拟器/脚本/变量结构/index.js';
```

### 步骤 4：验证实时修改流程

1. 确保酒馆地址 `http://127.0.0.1:8000` 上有 `神秘复苏模拟器` 角色卡
2. 确保酒馆助手 → 开发 → 实时监听 已启用
3. 点击编辑器右下角 **Go Live** 启动 Live Server（观察端口是否为 5500）
4. 终端运行 `pnpm watch`
5. 修改 `src/神秘复苏模拟器/` 下的文件，保存后酒馆应自动更新
6. 酒馆助手实时监听的 WiFi 图标应变绿

### 步骤 5：发布时还原

完成实时编写后：
1. 停止 `pnpm watch`（Ctrl+C）
2. 运行 `pnpm build` 生成最终打包结果
3. 将 `[界面]状态栏` 正则的 URL 改回 CDN 或使用 `pnpm build` 后的打包结果内联
4. 将变量结构脚本的 import 改回 CDN
5. 将打包结果同步到酒馆：`node tavern_sync.mjs push 神秘复苏模拟器 -f`

## 注意事项

- Live Server 端口如果不是 5500，步骤 2/3 中的端口号需相应调整
- `pnpm watch` 的打包结果体积是 `pnpm build` 的数十倍，仅用于开发，不要直接发布
- MVU 脚本（`import 'https://...bundle.js'`）是第三方库，不需要改为本地地址
- `.vscode/start-chrome-debug.cmd` 已在上一步改为 `http://127.0.0.1:8000`，无需再改
