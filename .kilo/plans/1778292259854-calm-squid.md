# 实时同步本地文件到酒馆 — 完整计划

## 目标
让本地文件（角色卡 YAML、世界书条目、脚本、界面等）的修改能实时同步到酒馆中，即改即生效。

---

## 一、架构说明（两条独立的同步链路）

### 链路 A：tavern_sync（角色卡/世界书/预设内容同步）
```
本地 YAML/文本文件变化 → tavern_sync watch (端口 6620) → WebSocket → 酒馆助手客户端脚本 → 酒馆 API 更新角色卡/世界书
```
- **触发时机**：修改 `src/神秘复苏模拟器/index.yaml` 或其引用的子文件（世界书条目、第一条消息等）
- **同步内容**：角色卡描述、世界书条目、正则、扩展字段等
- **服务端**：`node tavern_sync.mjs watch 神秘复苏模拟器 -f`
- **客户端**：酒馆助手全局脚本 `tavern_sync`（内容为 `import 'https://testingcf.jsdelivr.net/gh/StageDog/tavern_sync/dist/script.js'`）
- **启动方式**：`pnpm watch` 内部自动通过 `pnpm sync watch all -f` 启动子进程

### 链路 B：webpack watch（前端界面/脚本的 HMR）
```
本地 Vue/TS/CSS 文件变化 → webpack watch → 重新编译 → Socket.IO (端口 6621) → iframe 内 HMR 脚本 → location.reload()
```
- **触发时机**：修改 `src/神秘复苏模拟器/界面/状态栏/App.vue`、`global.css`、`store.ts` 等
- **同步内容**：状态栏界面的 HTML/CSS/JS
- **服务端**：webpack.config.ts 中的 `watch_tavern_helper` 钩子，在 `compiler.hooks.done` 时通过 Socket.IO 推送事件
- **客户端**：`index.html` 中内嵌的 Socket.IO 客户端脚本，监听 `message_iframe_updated` 事件后 `location.reload()`
- **启动方式**：`pnpm watch` 即 webpack watch 模式

### 两条链路的关系
- `pnpm watch` 是统一入口，同时启动 webpack watch 和 tavern_sync watch
- webpack watch 在第一次编译完成时，通过 `watch_tavern_sync` 钩子 spawn tavern_sync 子进程
- 两者完全独立，各自监听不同端口、处理不同文件

- 验证命令：`netstat -ano | findstr ":6620 :6621 :5500"` 应无输出

### 步骤 2：统一通过 `pnpm watch` 启动
- 在一个终端中运行 `pnpm watch`
- 确认日志中出现两条成功消息：
  - `[tavern_helper] 已启动酒馆监听服务`（端口 6621）
  - `[tavern_sync] 服务器成功连接到酒馆网页`（端口 6620）
- 确认无 `ELIFECYCLE Command failed` 或 `EADDRINUSE` 错误

### 步骤 3：验证 tavern_sync 连接（角色卡/世界书同步）
- 通过 Chrome DevTools MCP 检查酒馆控制台，确认 tavern_sync 客户端脚本已连接
- 应能看到类似 `[tavern_sync] 已连接` 或成功连接的日志
- 验证方式：通过 Chrome DevTools 检查 iframe 中 `window.tavern_sync` 是否存在，或检查酒馆主页面中 JS-Slash-Runner 创建的 iframe 内是否加载了 tavern_sync 脚本

### 步骤 4：端到端验证 — tavern_sync 链路

### 步骤 5：端到端验证 — webpack HMR 链路

### 步骤 7：确认日常使用流程

## 四、风险点与注意事项
1. **端口冲突**：6620 和 6621 端口必须空闲，否则 `pnpm watch` 会失败
2. **tavern_sync 客户端脚本必须启用**：酒馆助手全局脚本库中的 `tavern_sync` 脚本开关必须打开，否则酒馆无法接收服务端推送
3. **http-server 不再需要**：之前手动启动的端口 5500 的 http-server 仅用于测试，`pnpm watch` + tavern_sync 已覆盖完整链路
4. **index.html 中的 HMR 脚本与 tavern_sync 无关**：index.html 中的 Socket.IO 脚本连接的是端口 6621（webpack HMR），tavern_sync 连接的是端口 6620，两者互不干扰
5. **Windows Watchpack 警告**：`lstat 'D:\System Volume Information'` 警告可忽略，不影响功能
